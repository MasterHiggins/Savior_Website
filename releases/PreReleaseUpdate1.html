Pre Release 1 Results

Article from Chris Higgins
(Developer)
This article does not contain spoilers

The results of Pre release 1 have wrapped up. This was the most enoyable part of development so far, as it is mostly YouTube Shorts and playing the full game (Part 1, at least).

This article serves more as a report on the observations I made, so it may be technical.

There were 3 major issues this pre-release. 2 of them have been fixed for the next pre-release, starting this week.

The first issue was regarding the saving of Savior. When leaving the game to go to the menu, the game would save a file, but not at that moment, and instead during the 45 second session save that occurs, resulting in the game not saving if it hasn't been 45 seconds. This has been fixed.

The second issue was regarding loading a save file. When loading, the game would not load the file correctly, and it would be a mix of the save file and the loacl 45 second saves that generated from farming. This has been resolved for the next pre-release test.

The third issue is the loading of areas altered in game by in game options. For example, if you interact with an ancient module, and a rock is removed, then it may not be removed when loading. This is being worked on.

The game, otherwise, operated as I expected, and the next prerelease test should begin late this week, early next week.
